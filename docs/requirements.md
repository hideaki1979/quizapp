| 項目 | 内容 |

---

## 1. 概要

4択クイズに回答できる Web アプリケーション。クイズは**複数選択（複数正解あり）形式**で、すべての正解を過不足なく選択した場合のみ正解と判定する。

利用者を **管理者（Admin）** と **プレイヤー（Player）** の2レイヤーに分離し、それぞれ独立した画面・権限を持つ。

### 1.1 用語定義

| 用語 | 意味 |
| --- | --- |
| 管理者 | クイズ・カテゴリー・解説を管理する利用者。専用ログインで認証する。 |
| プレイヤー | クイズに回答する一般利用者。 |
| カテゴリー | クイズを分類する単位（例: 歴史、IT、スポーツ）。 |
| クイズ（問題） | 問題文・解説を持つ1問。1つのカテゴリーに属する。 |
| 選択肢 | クイズに紐づく回答の選択肢。正解フラグを持つ。1問につき4つ（4択）。 |
| 正解 | 1問に含まれるすべての正解選択肢を過不足なく選んだ状態。 |

## 2. レイヤー構成

```text
┌─────────────────────────┐      ┌─────────────────────────┐
│      管理者レイヤー         │      │     プレイヤーレイヤー       │
│   (Admin)               │      │   (Player)              │
├─────────────────────────┤      ├─────────────────────────┤
│ ・専用ログイン画面          │      │ ・クイズ出題・回答          │
│ ・管理ダッシュボード        │      │ ・結果表示・解説閲覧        │
│ ・カテゴリー管理            │      │                         │
│ ・クイズ／選択肢／解説 管理  │      │                         │
└─────────────────────────┘      └─────────────────────────┘
              │                              │
              └──────────┬───────────────────┘
                         ▼
              ┌─────────────────────┐
              │   共有データ層 (DB)    │
              │ 管理者/カテゴリー/      │
              │ クイズ/選択肢          │
              └─────────────────────┘
```

管理者画面とプレイヤー画面はルーティング・ミドルウェア・認証ガードを分離する。

## 3. 機能要件

### 3.1 管理者レイヤー

| ID | 機能 | 要件 |
| --- | --- | --- |
| A-01 | 管理者認証 | 管理者は**専用のログイン画面**でメールアドレスとパスワードにより認証する。未認証では管理画面にアクセスできない。 |
| A-02 | 管理ダッシュボード | 認証後、管理画面（ダッシュボード）にアクセスできる。 |
| A-03 | カテゴリー登録 | 管理者はクイズのカテゴリーを登録できる（カテゴリー名・説明）。 |
| A-04 | カテゴリー編集・削除 | 管理者は既存カテゴリーを編集・削除できる。カテゴリーを削除すると、所属するクイズおよびその選択肢も**カスケード削除**される（R-06）。 |
| A-05 | クイズ追加 | 管理者はクイズ（問題）を追加できる。問題文・カテゴリー・4つの選択肢・各選択肢の正解フラグを登録する。 |
| A-06 | クイズ編集 | 管理者は既存クイズの問題文・カテゴリー・選択肢・正解フラグを編集できる。 |
| A-07 | クイズ削除 | 管理者は既存クイズを削除できる。 |
| A-08 | 解説の追加 | 管理者はクイズに解説を追加・編集できる。 |

#### 管理者ログイン要件（A-01）詳細
- ログイン画面はプレイヤー側とは別 URL（例: `/admin/login`）。
- 認証成功で管理者セッションを確立し、管理画面へリダイレクト。
- 認証失敗時はエラーメッセージを表示。
- ログアウト機能を備える。

### 3.2 プレイヤーレイヤー

| ID | 機能 | 要件 |
| --- | --- | --- |
| P-01 | アカウント不要 | プレイヤーは**アカウントを作成せず**（匿名で）クイズに挑戦できる。 |
| P-02 | カテゴリー選択 | プレイヤーは挑戦するカテゴリーを選択できる。 |
| P-03 | ランダム出題 | 選択したカテゴリー内のクイズが**ランダムな順序**で出題される。各問は4つの選択肢を持つ。 |
| P-04 | 複数選択回答 | プレイヤーは1問につき複数の選択肢を選んで回答できる。 |
| P-05 | 正誤判定 | **すべての正解選択肢を過不足なく選択した場合のみ正解**と判定する。 |
| P-06 | 正誤・解説表示 | 回答すると、正解／不正解の判定と該当クイズの解説が表示される。 |
| P-07 | リザルト表示 | 選択したカテゴリー内のクイズを**すべて解答し終えると**リザルト画面を表示し、何問正解できたか（正解数 / 総問題数）を表示する。 |
| P-08 | リザルト後の導線 | リザルト画面から、**同じカテゴリーへの再挑戦**と**別カテゴリーの選択**ができる導線を提供する。 |

#### プレイヤー出題フロー（P-02〜P-07）詳細

```text
カテゴリー選択
   ↓
カテゴリー内の全クイズをランダム順で1問ずつ出題
   ↓
複数選択で回答 → 正誤判定（全正解一致のみ正解） → 正誤＋解説を表示
   ↓
（次の問題へ。カテゴリー内の全問終了まで繰り返し）
   ↓
リザルト表示（正解数 / 総問題数）
   ↓
「同じカテゴリーで再挑戦」 または 「別カテゴリーを選ぶ」
```

- 出題範囲は選択した**カテゴリー内の全クイズ**。1セッションでそのカテゴリーを完答する。
- 出題順はランダム。
- 回答結果はセッション内で集計し、完答時にリザルトとして表示する（DB への永続保存は行わない）。
- リザルトから再挑戦（同カテゴリーを再度ランダム出題）・別カテゴリー選択ができる（P-08）。

## 4. データモデル

`docs/er.drawio` および講義スライドの設計を参考に、命名はフレームワーク標準・講義準拠、型はベストプラクティスを採用。マイグレーションは本リポジトリ（Laravel 側）で作成する。詳細な DDL は [screen-design.md](screen-design.md) §8 参照。

### 4.1 エンティティ

#### users（管理者）

| カラム | 型 | 説明 |
| --- | --- | --- |
| id | PK | 識別子 |
| name | string | 名前 |
| email | string (unique) | メールアドレス（ログイン ID） |
| password | string | パスワード（ハッシュ） |

> 管理者認証は Laravel 公式 Starter Kit / Fortify 標準の `users` テーブルを利用する。

#### categories（カテゴリー）

| カラム | 型 | 説明 |
| --- | --- | --- |
| id | PK | 識別子 |
| name | string | カテゴリー名 |
| description | text | 説明 |

#### quizzes（クイズ／問題）

| カラム | 型 | 説明 |
| --- | --- | --- |
| id | PK | 識別子 |
| category_id | FK → categories | 所属カテゴリー |
| question | text | 問題文 |
| explanation | text | 解説 |

#### options（選択肢）

| カラム | 型 | 説明 |
| --- | --- | --- |
| id | PK | 識別子 |
| quiz_id | FK → quizzes | 所属クイズ |
| content | string | 選択肢のテキスト |
| is_correct | boolean | 正解フラグ（true: 正解 / false: 不正解）。MySQL 上は TINYINT(1) |

### 4.2 リレーション

```text
categories 1 ──< quizzes 1 ──< options
users（クイズ・カテゴリーを管理する管理者）
```

- カテゴリー : クイズ = 1 : N
- クイズ : 選択肢 = 1 : N（要件上は1問あたり4件 = 4択固定）
- 1問に正解フラグ `true` の選択肢が1つ以上存在しうる（複数正解形式）
- 外部キーは `ON DELETE CASCADE`：カテゴリー削除 → 所属クイズ削除 → その選択肢も削除（R-06）

## 5. 業務ルール

| ID | ルール |
| --- | --- |
| R-01 | クイズは必ず1つのカテゴリーに属する。 |
| R-02 | 1問の選択肢は4件（4択）とする。 |
| R-03 | 1問につき正解選択肢（`is_correct = true`）は1件以上。複数正解を許容する。 |
| R-04 | プレイヤーが選択した集合が、正解選択肢の集合と完全一致した場合のみ正解。 |
| R-05 | 解説はクイズに紐づき、回答後に表示する。 |
| R-06 | カテゴリーを削除すると、所属クイズおよびその選択肢もカスケード削除する（`categories → quizzes → options` の順に `ON DELETE CASCADE`）。 |

## 6. 非機能要件（初期方針）

| 区分 | 内容 |
| --- | --- |
| 認証 | 管理者は Fortify ベースの認証。パスワードはハッシュ保存。 |
| 権限分離 | 管理者用ルートは認証ミドルウェアで保護。プレイヤー用ルートと分離。 |
| 入力検証 | フォームリクエストでバリデーション（必須・型・選択肢4件・正解1件以上など）。 |
| UI | Inertia + React + TypeScript による SPA 的画面遷移。 |
| DB | MySQL（外部キー制約あり）。 |

## 7. 確定事項（旧・未確定事項）

すべての論点が確定済み。

| 区分 | 状態 | 内容 |
| --- | --- | --- |
| プレイヤーの認証要否 | ✅ 確定 | アカウント不要・匿名で挑戦可能（P-01）。 |
| 出題ロジック | ✅ 確定 | 選択カテゴリー内の全クイズをランダム順で出題（P-02〜P-03）。 |
| 出題範囲 | ✅ 確定 | カテゴリー内完答。カテゴリー横断出題は行わない。 |
| 正誤判定 | ✅ 確定 | 完全一致のみ正解。部分点なし（R-04）。 |
| スコア・回答履歴の保存 | ✅ 確定 | 保存しない。ランキング・成績記録は対象外。回答結果はセッション内集計・完答時リザルト表示のみ（`players` / `answers` テーブルは追加しない）。 |
| 選択肢数の固定/可変 | ✅ 確定 | 4択固定（R-02）。 |
| リザルト後の遷移 | ✅ 確定 | 同カテゴリー再挑戦・別カテゴリー選択の導線あり（P-08）。 |
| カテゴリー削除時の挙動 | ✅ 確定 | カスケード削除（カテゴリー→クイズ→選択肢、R-06）。 |
